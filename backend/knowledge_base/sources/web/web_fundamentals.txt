HTTP AND REST
HTTP (HyperText Transfer Protocol) is the foundation of data communication on the web.

HTTP methods:
GET: retrieve a resource. Idempotent, no body.
POST: create a resource. Not idempotent, has body.
PUT: replace a resource entirely. Idempotent.
PATCH: partially update a resource. Not necessarily idempotent.
DELETE: remove a resource. Idempotent.

HTTP status codes:
2xx Success: 200 OK, 201 Created, 204 No Content.
3xx Redirection: 301 Moved Permanently, 304 Not Modified.
4xx Client Error: 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 422 Unprocessable Entity.
5xx Server Error: 500 Internal Server Error, 502 Bad Gateway, 503 Service Unavailable.

REST (Representational State Transfer): architectural style for distributed systems. Constraints: stateless, client-server, cacheable, uniform interface, layered system.
RESTful APIs: resources identified by URLs, manipulated using HTTP methods, responses in JSON or XML.

DATABASES AND SQL
Relational databases store data in tables with rows and columns. SQL (Structured Query Language) is used to query and manipulate data.

ACID properties: Atomicity (all or nothing), Consistency (valid state), Isolation (concurrent transactions don't interfere), Durability (committed changes persist).

SQL operations: SELECT, INSERT, UPDATE, DELETE. JOINs: INNER, LEFT, RIGHT, FULL OUTER. Aggregations: GROUP BY, HAVING, COUNT, SUM, AVG.

Indexes: speed up read queries but slow down writes. B-tree index is most common. Use indexes on columns frequently used in WHERE, JOIN, ORDER BY.

N+1 query problem: loading N related records with N+1 separate queries. Solution: eager loading (JOIN or prefetch).

NoSQL databases: MongoDB (document), Redis (key-value), Cassandra (wide-column). Trade ACID for scalability and flexibility. Use when schema is flexible or data is unstructured.

AUTHENTICATION AND SECURITY
JWT (JSON Web Token): stateless authentication. Contains header, payload, signature. Server issues token on login; client sends it in Authorization header. Verify signature on each request without DB lookup.

OAuth 2.0: authorization framework. Allows third-party access without sharing passwords. Flows: Authorization Code (web apps), Client Credentials (server-to-server), Implicit (deprecated).

Password security: never store plain text passwords. Use bcrypt, argon2, or scrypt. These are slow by design to resist brute-force attacks.

Common vulnerabilities:
SQL Injection: attacker inserts malicious SQL. Prevention: parameterized queries / ORM.
XSS (Cross-Site Scripting): inject malicious scripts into web pages. Prevention: escape output, Content Security Policy.
CSRF (Cross-Site Request Forgery): trick user into submitting malicious request. Prevention: CSRF tokens, SameSite cookies.
CORS (Cross-Origin Resource Sharing): browser policy restricting cross-origin requests. Configured on the server with allowed origins.

REACT AND FRONTEND
React is a JavaScript library for building component-based user interfaces.

Virtual DOM: React maintains a lightweight in-memory representation of the DOM. On state change, React diffs the virtual DOM and applies minimal real DOM updates (reconciliation).

Hooks:
useState: manage local component state.
useEffect: side effects (data fetching, subscriptions). Cleanup function runs on unmount.
useContext: consume React context without prop drilling.
useReducer: complex state logic, alternative to useState.
useMemo / useCallback: memoize values and functions to prevent unnecessary re-renders.

State management: for global state, use Context API (simple), Zustand (lightweight), or Redux (complex apps).

Next.js: React framework with server-side rendering (SSR), static site generation (SSG), and API routes. File-based routing. Use SSR for SEO-critical pages, SSG for static content.

SYSTEM DESIGN BASICS
Scalability: ability to handle increased load.
Vertical scaling: add more resources to existing server. Limited ceiling.
Horizontal scaling: add more servers. Requires load balancer and stateless design.

Load balancer: distributes traffic across servers. Algorithms: round robin, least connections, IP hash.

Caching: store frequently accessed data in fast storage (Redis, Memcached). Cache-aside pattern: app checks cache first, loads from DB on miss, stores in cache.
Cache invalidation strategies: TTL (time-to-live), write-through, write-behind, cache-aside.

CDN (Content Delivery Network): serve static assets from servers geographically close to users. Reduces latency.

Message queues (RabbitMQ, Kafka): decouple services, handle async processing, smooth traffic spikes.

Database sharding: split data across multiple DBs by a shard key. Increases write throughput but complicates joins and transactions.

Microservices: decompose application into small, independently deployable services. Benefits: independent scaling, fault isolation. Drawbacks: distributed systems complexity, network overhead.
