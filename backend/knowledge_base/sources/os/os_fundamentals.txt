PROCESSES AND THREADS
A process is an instance of a program in execution. It has its own memory space (code, data, heap, stack), file descriptors, and OS resources. A thread is a lightweight unit of execution within a process, sharing memory and resources with other threads in the same process.

Process states: new, ready, running, waiting, terminated.
Context switching: saving the state of a running process and restoring another. Involves saving/restoring registers, program counter, and memory mappings. Expensive due to cache invalidation.

Inter-process communication (IPC): pipes, message queues, shared memory, semaphores, sockets. Shared memory is fastest; pipes are simplest.

Threads vs Processes: threads share memory (faster communication, but race conditions possible); processes are isolated (safer, but IPC overhead). Use threads for I/O-bound tasks, processes for CPU-bound tasks.

PROCESS SCHEDULING
The CPU scheduler decides which process/thread runs next. Goals: maximize CPU utilization, throughput, fairness; minimize waiting time, turnaround time, response time.

Scheduling algorithms:
FCFS (First Come First Served): non-preemptive, simple, convoy effect.
SJF (Shortest Job First): optimal average waiting time, but requires knowing burst time.
Round Robin: preemptive, each process gets a time quantum. Fair, good for time-sharing. Performance depends on quantum size.
Priority Scheduling: higher priority runs first. Starvation possible — solved by aging.
Multilevel Queue: separate queues for different process categories.
Multilevel Feedback Queue: processes move between queues based on behavior. Most flexible.

MEMORY MANAGEMENT
Virtual memory: each process has its own virtual address space, mapped to physical memory by the OS. Allows processes larger than physical RAM.

Paging: divide memory into fixed-size pages (virtual) and frames (physical). Page table maps virtual pages to physical frames. Eliminates external fragmentation, but has internal fragmentation.

Page replacement algorithms:
FIFO: replace the oldest page. Simple, but Belady's anomaly possible.
LRU (Least Recently Used): replace the page unused longest. Good approximation of optimal.
Optimal: replace the page that won't be used for longest time. Theoretical benchmark only.

Thrashing: excessive paging when a process has insufficient frames, spending more time paging than executing. Solved by working set model or page fault frequency.

Translation Lookaside Buffer (TLB): hardware cache for page table entries. Speeds up virtual-to-physical translation. TLB miss forces a page table walk.

DEADLOCKS
A deadlock occurs when processes are permanently blocked, each waiting for a resource held by another.

Four necessary conditions (Coffman): mutual exclusion, hold and wait, no preemption, circular wait.

Deadlock handling strategies:
Prevention: eliminate one of the four conditions.
Avoidance: Banker's algorithm — only grant requests if system stays in safe state.
Detection and recovery: allow deadlocks, detect via resource allocation graph, recover by killing processes or preempting resources.
Ignorance: most OS use this (ostrich algorithm) — deadlocks are rare enough to ignore.

SYNCHRONIZATION
Race condition: two threads access shared data concurrently, result depends on execution order.
Critical section: code that accesses shared resources. Must be mutually exclusive.

Mutex (mutual exclusion lock): only one thread holds it at a time. Binary semaphore.
Semaphore: integer counter. wait() decrements (blocks if 0), signal() increments.
Monitor: high-level synchronization construct with condition variables.

Classic problems: Producer-Consumer (bounded buffer), Readers-Writers, Dining Philosophers.

Spinlock: busy-waits for the lock. Efficient for short waits on multicore systems; wasteful otherwise.

FILE SYSTEMS
A file system organizes data on storage devices. Common file systems: ext4 (Linux), NTFS (Windows), APFS (macOS), FAT32.

Inode: data structure storing file metadata (size, permissions, timestamps, data block pointers). Does not contain the filename.
Directory: maps filenames to inode numbers.

File allocation methods:
Contiguous: fast sequential access, external fragmentation.
Linked: no fragmentation, poor random access.
Indexed: inode holds block pointers, good random access.

Journaling: log changes before applying them. Ensures consistency after crashes. ext4 and NTFS use journaling.
